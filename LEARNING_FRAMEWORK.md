Learning Framework

Purpose

The goal of learning is not to memorize information.

The goal is to discover the smallest set of abstractions that explains the largest amount of reality.

Facts fade.
Commands change.
Technologies evolve.

Well-chosen mental models continue generating correct predictions long after the details have been forgotten.

⸻

Learning Protocol

1. Why does this exist?

Before learning how something works, understand why it had to exist.

Questions:

* What problem was this created to solve?
* What limitations existed before it?
* What trade-offs does it introduce?
* What would happen if it didn’t exist?

⸻

2. What problem does it solve?

State the problem in one or two sentences.

Avoid implementation details.

If the problem cannot be clearly articulated, understanding is likely incomplete.

⸻

3. What is the simplest abstraction?

Find the single mental model that explains most of the domain.

This is the compression algorithm.

Examples:

* Git → Immutable graph of commits with movable references.
* SQL → Declarative language describing what data is wanted, not how to retrieve it.
* TCP/IP → Reliable communication between computers through packets.

A good abstraction should generate many correct predictions.

⸻

4. What analogy compresses the entire domain?

Translate the abstraction into an intuitive analogy.

A good analogy dramatically reduces cognitive load.

Examples:

* Git → A book with permanent pages and movable bookmarks.
* Branch → A bookmark pointing to a different storyline.
* Commit → A permanent snapshot in the book.

The analogy should make the system feel obvious rather than mysterious.

⸻

5. What vocabulary follows naturally?

Only after the abstraction is understood should terminology be introduced.

Vocabulary should describe the abstraction rather than replace it.

For each new term ask:

* What role does it play?
* How does it relate to the core abstraction?
* Could I explain it using the analogy?

⸻

6. What commands, formulas or tools operate on the abstraction?

Commands are verbs acting on the underlying model.

Examples:

* Git → commit, switch, merge, push, pull
* SQL → SELECT, FROM, WHERE, GROUP BY
* Physics → Equations describing relationships within the model

Understanding the model should make the commands intuitive.

⸻

7. Build intuition before memorization

Do not memorize isolated facts.

Instead:

* Predict behavior.
* Explain outcomes.
* Draw diagrams.
* Invent analogies.
* Ask “What do I expect to happen?”

When predictions consistently match reality, understanding has been achieved.

⸻

8. Practice deliberately

Only after intuition exists should repetition begin.

Practice should reinforce understanding rather than compensate for its absence.

Examples:

* Write code.
* Solve problems.
* Build projects.
* Teach the concept.
* Review mistakes.
* Iterate.

⸻

Indicators of Deep Understanding

A concept is considered understood when it can be:

* Explained simply.
* Predicted before testing.
* Connected to other concepts.
* Taught using an original analogy.
* Reconstructed after forgetting details.

⸻

Guiding Principles

* Seek understanding before memorization.
* Search for compression, not accumulation.
* Build schemas before vocabulary.
* Learn from first principles whenever possible.
* Prefer generative mental models over isolated facts.
* Every new concept should connect to an existing one.
* Every abstraction should reduce cognitive load.
* Every learning session should leave behind reusable knowledge.

⸻

The Learning Loop

Learning is complete when the abstraction becomes generative—when it allows accurate reasoning about situations that have never been encountered before.
